
# Game Design Document: Triviape

## 1. Core Loop

The core gameplay loop of Triviape is centered around the "Daily Quiz." The loop is designed to be simple, engaging, and repeatable, encouraging users to come back every day.

1.  **Discover:** The user logs in and sees the new Daily Quiz for the day.
2.  **Play:** The user plays through the quiz, answering a series of questions.
3.  **Review:** The user finishes the quiz and reviews their results (score, correct answers, rewards).
4.  **Compete:** The user checks their ranking on the daily leaderboard.
5.  **Repeat:** The user returns the next day for a new quiz.

## 2. Game Mechanics

### 2.1. Daily Quiz

*   **Frequency:** A new quiz is available every 24 hours.
*   **Content:** The quiz consists of a mix of questions from different categories and difficulty levels.
*   **Structure:** The quiz has a fixed number of questions.
*   **Time Limit:** Each question may have a time limit to encourage quick thinking.

### 2.2. Scoring

*   **Correct Answers:** Users earn points for each correct answer.
*   **Difficulty Bonus:** More points may be awarded for correctly answering more difficult questions.
*   **Time Bonus:** Users may earn bonus points for answering questions quickly.

### 2.3. Progression

*   **Experience Points (XP):** Users earn XP for completing quizzes. The amount of XP earned is based on their score.
*   **Levels:** Users level up by earning XP. Each level requires a certain amount of XP to reach.
*   **Coins:** Users earn coins for completing quizzes and leveling up. Coins can be used to unlock new features or content in the future.

### 2.4. Leaderboards

*   **Daily Leaderboard:** The primary leaderboard ranks players based on their score in the Daily Quiz.
*   **Other Leaderboards:** The system is designed to support other types of leaderboards in the future (e.g., weekly, monthly, all-time, by category).

### 2.5. Social Mechanics

*   **Friends:** Users can add friends to see their progress and compete with them on the leaderboards.
*   **Challenges:** (Future) Users will be able to challenge their friends to a quiz.

### 2.6. Multiplayer

*   **Real-time Competition:** Users can join multiplayer games to compete against each other in real-time.
*   **Game Rooms:** Players can create or join game rooms.
*   **Live Results:** Players can see each other's progress and scores in real-time.

## 3. User Experience

### 3.1. Onboarding

*   **Simplicity:** The onboarding process is simple and straightforward. New users can quickly sign up and start playing their first quiz.
*   **Tutorial:** A brief tutorial guides new users through the main features of the application.

### 3.2. Game Interface

*   **Clarity:** The quiz interface is clean and easy to understand. Questions and answers are clearly displayed.
*   **Feedback:** Users receive immediate feedback on their answers (correct or incorrect).
*   **Responsiveness:** The interface is fast and responsive, with smooth transitions and animations.

### 3.3. Visual and Audio Design

*   **Visual Style:** The application has a modern and playful visual style.
*   **Animations:** Animations are used to enhance the user experience and provide feedback.
*   **Sound Effects:** (Future) Sound effects will be used to provide feedback and make the experience more immersive.
